HeadAnalyser

v1.1.1 v2 in development

Desktop application for groundwater hydraulic gradient analysis — estimating flow direction and magnitude from borehole head measurements using triangulation-based methods.

Python PyQt5 NumPy / SciPy Matplotlib Folium / Leaflet pyproj Tkinter
Overview

Release build overview from the published HeadAnalyser application.

Description

HeadAnalyser calculates hydraulic gradients (magnitude and direction) from spatial measurements of water table elevations at multiple boreholes. The underlying method is based on least-squares fitting across all possible triangles formed by data points — an approach derived from the work of Devlin & Schillig (2017).

The program was developed at DTU Sustain (Technical University of Denmark) under the supervision of Prof. Poul L. Bjerg, with contributions from Grégory Lemaire and Anton Bo Bøllingtoft. It is published on DTU Data under a BSD 3-Clause licence and has been downloaded 421 times as of May 2025. It is actively used in DTU courses and by industry practitioners.

Highlights
Gradient vector field
Displays hydraulic gradient magnitude and direction at each triangle centroid. Arrow length represents magnitude, colour maps to head value, and vectors can be downsampled automatically for dense datasets.
Interactive web mapping
Renders borehole locations on satellite or OpenStreetMap tiles. The V2 work expands this with contour overlays, heatmaps coloured by rejection reason, and external GeoJSON/WMS support within the application.
Directional analysis
Includes rose diagrams and circular statistics for analysing dominant groundwater flow directions, angular spread, and coherence in the calculated gradient field.
Automated reporting
Generates PDF reports containing statistics, selected plots, location maps, gradient result tables, and rejected triangle summaries with configurable output sections.

Key Features
Triangulation-based gradient estimation
Generates all possible triangles from borehole data points and applies least-squares fitting to calculate hydraulic gradient magnitude and direction for each. Quality checks reject triangles with extreme geometry or insufficient head uncertainty. Based on the methodology in Devlin & Schillig, Hydrogeology Journal (2017).
Multi-type visualisation
2D contour scatter plots, 3D interpolated surfaces, gradient vector fields, directional rose diagrams, gradient histograms, borehole depth plots, and triangle overlay views — all rendered with Matplotlib and fully customisable (colormaps, label styles, marker sizes, grid/compass overlays).
Interactive web mapping
Embedded Leaflet-based maps (via Folium) displaying measurement points on OpenStreetMap / satellite basemaps. V2 adds contour overlays, heatmaps, vector field rendering, triangle overlays, and external GeoJSON/WMS layer support directly within the application.
Coordinate system handling
Automatic UTM32 EUREF89 (EPSG:25832) to WGS84 (EPSG:4326) conversion via pyproj — essential for placing Danish borehole data onto web maps and for cross-referencing with external geospatial datasets.
Automated PDF reporting
One-click generation of comprehensive PDF reports including statistics, selected plots, location maps, gradient result tables, and rejected triangle summaries. Built with ReportLab and Selenium for map rendering.
Sensitivity analysis V2
Multi-parameter sweep engine for exploring how gradient results respond to changes in head uncertainty, confidence level, and triangle quality constraints. Produces statistical summaries with circular mean for directional data.
Geo.dk geological integration V2
Integration with the Danish national geological data service (Geo.dk) for retrieving geological cross-sections. Overlays borehole positions onto SVG stratigraphy profiles with coordinate mapping between UTM32 and the Geo.dk SVG coordinate space.
Flexible data import
Reads CSV (with automatic delimiter detection), Excel (.xlsx with sheet selection), and JSON files. User-configurable column mapping for ID, coordinates, hydraulic head, and optional depth columns. Handles comma/dot decimal separators automatically.

Python PyQt5 PyQtWebEngine NumPy / SciPy Matplotlib Folium / Leaflet pyproj ReportLab
Overview

System overview diagram for the current modular rewrite.

Description

HeadAnalyser V2 is a complete rewrite of the original HeadAnalyser, moving from a monolithic Tkinter script (~4,500 lines) to a modular PyQt5 application with clean separation of core logic, UI, and styling.

The core gradient estimation engine is unchanged — the same triangulation-based method from Devlin & Schillig (2017) — but the interface, architecture, and feature set are significantly expanded. V2 introduces multi-dataset tabs, embedded web maps rendered via PyQtWebEngine, a sensitivity analysis engine, and integration with the Danish national geological data service (Geo.dk).

What's New in V2
Modern PyQt5 interface
Custom frameless window chrome with a four-panel layout, runtime theme switching, high-DPI support, and multi-tab handling for simultaneous dataset analysis.
Embedded web mapping
Leaflet maps rendered inside the application via PyQtWebEngine and Folium, with contour overlays, heatmaps, vector fields, triangle visualisation, and support for external GeoJSON and WMS layers.
Sensitivity analysis experimental
Multi-parameter scenario sweeps for testing how gradient results respond to uncertainty assumptions, confidence levels, and triangle quality constraints.
Geo.dk geological integration experimental
Integration with the Danish national geological data service for retrieving cross-sections and aligning borehole locations with SVG-based stratigraphy outputs.

GrainSizeAnalysis

v0.9.0-beta

Desktop application for grain size distribution analysis and hydraulic conductivity estimation — a full Python reimplementation of the Excel-based HydrogeoSieveXL tool, with 16 validated K-calculation methods and integrated report generation.

Python PyQt6 NumPy / Pandas Matplotlib ISO 14688 USCS / ASTM D2487 ReportLab
Overview

Workflow overview diagram for the current beta application.

Description

GrainSizeAnalysis is a PyQt6 desktop application for processing sieve and hydrometer data from soil and sediment samples. Its primary purpose is to compute hydraulic conductivity (K) from grain-size distributions using 16 peer-reviewed empirical methods — covering the range from classical Hazen (1892) and Terzaghi to Kozeny-Carman and Alyamani-Sen — all validated against the Excel VBA reference implementation HydrogeoSieveXL (Devlin group).

The program supports both ISO 14688-1:2017 (Danish/European standard) and USCS/ASTM D2487-17 (American standard) classification schemes for grain fraction determination and soil type identification, with the option to define custom grain-size boundaries. It is developed in the context of DTU Sustain (Prof. Poul L. Bjerg's group) to modernise field data workflows for contaminated site investigations.

Highlights
Grain-size distribution curves
Semi-log plots with ISO and USCS classification support, D-values, Cu, Cc, and the ability to compare individual or multiple samples within the same workspace.
16 hydraulic conductivity methods
Simultaneous K-estimation across the implemented methods with applicability checks, literature references, and unit switching between environmental engineering conventions.
Multi-sample comparison
Dedicated comparison tooling for side-by-side analysis of multiple datasets, including overlaid curves, K-method summaries, grain fractions, and statistical summaries.
Report generation
Report composer with in-app HTML preview and export paths for PDF, DOCX, and raw figure or table outputs.

Key Features
16 validated K-calculation methods
Implements all major empirical and theoretical hydraulic conductivity methods from the literature: Hazen (1892 and simplified), Slichter, Terzaghi, Beyer, Sauerbrei, Kruger, Kozeny-Carman, Zunker, Zamarin, USBR, Barr, Alyamani-Sen, Chapuis, Shepherd, and Krumbein-Monk. All implementations are validated against the Excel VBA reference tool HydrogeoSieveXL — 15 of 16 methods achieve <1% error; Krumbein-Monk has a known geometric mean discrepancy. Reference: Devlin (2015), Hydrogeology Journal · Vukovic & Soro (1992).
Dual classification standard support
Grain fraction percentages (clay/silt/sand/gravel/cobble) and soil classification are computed under either ISO 14688-1:2017 (European/Danish standard) or USCS/ASTM D2487-17 (US standard), using log-linear interpolation along the gradation curve at each scheme boundary. For USCS, well-graded/poorly-graded designations (SW, SP, GW, GP) are derived from Cu and Cc. Custom grain-size boundaries can be defined for non-standard schemes.
Flexible data import with column mapping
Reads CSV (automatic delimiter detection) and Excel files (.xls, .xlsx), including multi-sheet workbooks — a single dialog handles all files sharing the same sheet structure, with each sheet treated as a separate sample. A column mapping dialog resolves non-standard column headers, with pattern learning and batch application so one mapping can be reused across hundreds of files.
Interactive plot workspace
Matplotlib-based plot widget with controls for colourmap, label styles, gridlines, and classification zone visibility. Distribution curves are plotted on a log x-axis with D-value markers (D10, D50, D60, etc.). K bar charts use a log y-axis with per-bar applicability annotations. All plots are rendered by a widget-independent renderer module that also feeds headless report export.
Multi-sample comparison tab
Side-by-side comparison of 2 or more datasets across three sub-views: overlaid distribution plots, K-method summary table per dataset, and statistical comparison. Useful for comparing grain-size parameters across borehole depths or site zones. Dataset selection dialog allows ad-hoc subset selection without closing loaded data.
Report composer and export
Accordion-based report composer for building professional analysis documents. Sections are togglable (distribution plots, K-results, statistics, classification, sample metadata). HTML preview is rendered in-app via PyQtWebEngine. Export options: PDF (browser print), DOCX (python-docx), and raw table/plot export from the dedicated Export tab.
Porosity and temperature inputs
Most K-methods require effective porosity (n) and water temperature (°C) as inputs. The application allows per-sample overrides, with Urumovic-based calculated porosity as a default. Temperature correction follows Vukovic & Soro (1992) polynomial viscosity model. Results can be displayed in m/s, cm/s, m/d, cm/min, or ft/d.

Regionernes overfladevand

Automated data pipeline and QGIS cartography for assessing groundwater contamination impacts on Danish regulated surface waters — commissioned by Miljøstyrelsen.

Background

Following a 2014 amendment to Denmark's soil contamination law, the five regions conducted systematic risk screenings of roughly 36,000 mapped contaminated sites. In 2021–22, around 400 V1/V2 classified sites near regulated streams and coastal waters were sampled for environmentally hazardous substances (MFS). DTU Sustain was commissioned by Miljøstyrelsen to develop a method for evaluating the chemical impact of groundwater bodies on these regulated surface waters. The monitoring data from the regions formed the empirical foundation, covering compound groups including PFAS, pesticides, iron, and ammonium-N.

Maps
01 / 04
Contamination status — national overview

Binary classification of confirmed contamination (Ja / Nej / Uafklaret) at all stream monitoring stations across Denmark. Red points indicate confirmed surface water impact from a nearby contaminated site; green indicate no detected exceedance.

02 / 04
Industry type breakdown

Monitoring stations symbolised by industry type (landfill, dry-cleaner, metalworks, machinery workshop, fire-training ground, etc.) and contamination status. Enables identification of which industries are most frequently associated with confirmed surface water impact.

03 / 04
V1/V2 contaminated site register

All V1 (suspected, blue) and V2 (confirmed, red) mapped contaminated sites in Denmark, overlaid on the stream network. This national dataset from DKjord was integrated with the monitoring results to link contamination source polygons to individual sampling points.

04 / 04
Per-industry detail map — Syddanmark landfills

Example of a detail map generated for a specific region and industry type. Each symbol represents a compound group detected at the site (PFAS, landfill compounds, BTEX/hydrocarbons, pesticides, tar compounds). Point displacement renders overlapping multi-compound detections at shared monitoring locations. Black = primarily from site; red = site plus other sources.

What I built
Data preprocessing pipeline
A Python pipeline (pandas + GeoPandas) to load, clean, and filter the regions' monitoring datasets. Handles four compound groups — PFAS, pesticides, iron, and ammonium-N — and applies Danish regulatory thresholds to categorise each measurement: exceeding 10× the limit value, above the limit value, detected above the detection limit, or below the detection limit. Includes logic to resolve station identifiers from location names and select the highest-priority result per measurement point.
V1/V2 contaminated site integration
Joined national V1 (suspected) and V2 (confirmed) contaminated site polygons from DKjord to the monitoring point dataset via GeoPandas spatial joins, enabling map overlays linking contamination source areas to surface water sampling results.
PyQGIS automated map generation
A MapGenerator class in PyQGIS that iterates over all monitoring stations and compound groups to programmatically generate, style, and export individual maps. Each map includes categorised symbology, V1/V2 polygon overlays, the stream network layer, an inset overview map, a customised legend, and a scale bar — exported as 300 DPI PNG files organised by region, station, and compound group. Layers with only sub-detection-limit results are skipped automatically.
Categorised and displaced point rendering
PyQGIS scripting using QgsCategorizedSymbolRenderer to apply the four-tier exceedance colour scheme per compound, and QgsPointDisplacementRenderer to visually separate overlapping points at shared monitoring locations — critical for stations where multiple compounds were sampled.
Outputs
National overview maps by contamination status and industry type across all five Danish regions
Automated per-station, per-compound maps covering PFAS, pesticides (0.1 µg/l), iron (0.5 mg/l), and ammonium-N (0.78 mg/l)
Spatial integration of ~36,000 V1/V2 contaminated site polygons with surface water monitoring points
300 DPI map exports feeding into DTU Sustain's assessment reports delivered to Miljøstyrelsen

InSa-Drikkevand — PFAS & Pesticides in Groundwater

Co-authored Bilag A for the InSa-Drikkevand consortium — comparing national GEUS groundwater monitoring data against 7 Danish water utilities to characterise pesticide and PFAS contamination patterns.

Background

InSa-Drikkevand is a research consortium of major Danish water utilities — including HOFOR, NOVAFOS, Aarhus Vand, Aalborg Forsyning, DIN Forsyning, TREFOR, and VandCenterSyd — investigating the persistence and temporal development of PFAS and pesticide contamination in drinking water sources. GEUS had published national overviews of groundwater monitoring results across Denmark using data from the Jupiter database. The goal of this work was to compare each utility's own borehole monitoring results against the national picture, identifying where individual utilities diverge from the national average and for which compounds.

Analysis outputs
01 / 04
Depth distribution — national and per-utility

Depth distribution plots showing what percentage of screened intakes at each depth interval test positive for pesticides or PFAS (above DG or KV). Pesticide exceedances concentrate in the upper 20–40 m; PFAS detections are similarly shallow. Each utility plot is generated alongside the national GEUS baseline for direct comparison.

02 / 04
Per-utility frequency deviation

Bar charts showing the percentage-point difference between each utility's detection rate and the national average, per compound. DIN Forsyning shows notably elevated rates for desphenyl chloridazon (DPC: 33.3% vs. 14.2% nationally) and R471811 (15.6% vs. 1.8% nationally). Negative values indicate fewer detections than the national mean.

03 / 04
National baseline frequency (GEUS)

Replicated version of the GEUS national frequency plot, generated from the Jupiter extract using our own analysis code. Produced as a quality-control check to validate that our methodology matches GEUS's published results before applying the same approach to utility-level subsets. 9 key pesticides analysed for the period January 2018 – March 2023; 22+ PFAS compounds for September 2021 – March 2023.

04 / 04
Temporal pumping analysis — Staurbyskov (TREFOR)

Monthly water abstraction volumes per individual borehole at the Staurbyskov wellfield, operated by TREFOR (DIN Forsyning). Bar charts per borehole combined with a site overview map in a single figure layout using Pillow. Produced for three wellfields — Staurbyskov, Engkilde (Aalborg Forsyning), and Aike (DIN Forsyning) — to support temporal trend analysis at contaminated production sites.

What I built
Jupiter data extraction pipeline
Python/pandas pipeline to query and process groundwater monitoring records from GEUS's Jupiter database for 7 utilities, covering 65–265 boreholes per utility. Handled data quality issues including missing INTAKE records, boreholes absent from the extract despite being reported by utilities, and data approved only beyond the analysis period. Both time windows were applied consistently: pesticides (1 Jan 2018 – 23 Mar 2023) and PFAS (1 Sep 2021 – 23 Mar 2023).
Frequency table replication and validation
Code to generate hyppighedstabeller (frequency tables) for each utility, tabulating total intakes, detections above DG (drinking water guideline), and exceedances above KV (quality criterion) for each compound. Outputs were validated side-by-side against GEUS's published national tables in Bilag 1 to confirm methodological consistency before applying the analysis to individual utilities.
Depth distribution analysis
Plots showing contaminant detection rates (% above DG / above KV / not detected) stratified by screen depth in 10 m intervals, for both national data and each utility. Used INTAKE table depth data from Jupiter, with fallbacks to borehole casing depth and LITHOLOGI table chalk top values when screen data was missing — increasing data coverage for the depth plots.
Cross-utility compound comparison
Bar charts and UpSet-style plots comparing the top 10 compounds found above DG or KV across all 7 utilities simultaneously, visualising which compounds are widespread vs. utility-specific. DMS and 2,6-Dichlorobenzamide were found above DG in all 7 utilities; PFOA, SUM4, and SUM12 appeared above DG in 6 of 7.
Outputs
Frequency tables for 9 key pesticides and 22+ PFAS compounds per utility and nationally, validated against GEUS methodology
Depth distribution plots for all 7 utilities (DIN Forsyning, TREFOR, HOFOR, NOVAFOS, VandCenterSyd, Aalborg Forsyning, Aarhus Vand)
Cross-utility compound prevalence charts identifying which contaminants appear across multiple utilities vs. locally

Background

This project is carried out in close collaboration between DTU Sustain, GEUS, and Miljøstyrelsen. My work extends an earlier SQL-based calculation track into modular Python workflows that support parts of the risk assessment, including affected-volume and source-pressure style calculations based on groundwater chemistry, DKJord V1/V2 source areas, airport-related land-use polygons, groundwater-body geometries, and DK-model depth and thickness rasters. These contributions feed into the broader collaborative risk assessment, after which the final input file of groundwater bodies in risk is used as input to the tilstandsvurdering workflow I have implemented. The project currently runs as three separate tracks: PFAS Sum-4, PFAS Sum-22, and pesticides.

Workflow & visual outputs
01 / 03

Sedimentary Microbial Fuel Cells

Bachelor thesis investigating how Sedimentary Microbial Fuel Cells (SMFCs) can be used to remediate coastal marine sediments suffering from hypoxia and toxic sulfide accumulation — awarded Årets bachelorprojekt 2022 by the DTU Sustain Colding Award.

Background

Eutrophication from agricultural runoff and wastewater discharge causes algal blooms that lead to hypoxic and anoxic "dead zones" in coastal marine environments. Under these conditions, sulfide — a toxic compound — accumulates in marine sediments and diffuses into the water column, with damaging consequences for marine ecosystems. This thesis investigated Sedimentary Microbial Fuel Cells (SMFCs) as a bioremediation technology. An SMFC exploits naturally occurring cable bacteria, which can connect sulfide oxidation in anoxic sediment to oxygen reduction at the water surface through long-distance electron transport — effectively generating an electrical current while oxidising the harmful sulfide.

Experiments & findings
Experiment 1 — Activated carbon at the anode
Compared SMFCs with activated carbon electrodes (C) against steel-only electrodes (S). Carbon SMFCs produced higher stabilised electrical current and achieved significantly lower sulfide concentrations, attributed to activated carbon's large, porous surface area providing more sites for biofilm formation and thus a higher concentration of cable bacteria.
Experiment 2 — Cathode-to-anode ratio
Compared a 1:1 cathode-to-anode configuration (LL) with a smaller cathode variant (SL). LL configurations reached higher peak current density and higher sulfide turnover. The cathode was identified as the limiting factor for overall SMFC performance, consistent with oxygen reduction rate at the cathode being a general bottleneck for SMFC technology.
Analysis & data work
All data analysis was implemented in Python, including carbon and sulfide turnover calculations, correlation analysis (Pearson and Spearman rank), internal resistance estimation, and polarisation and power density curves. The data pipeline covered both reactor-scale and box-scale experimental setups with multiple replicates.

Background

With the global population expected to reach 9.8 billion by 2050, indoor vertical farming is gaining importance as a controlled, resource-efficient alternative to outdoor agriculture. UV-A radiation (315–400 nm) has shown potential to increase both crop yield and the accumulation of health-beneficial secondary metabolites — such as flavonoids and carotenoids — but dose-response relationships for specific crops and intensities remained poorly characterised. This project investigated how supplementing standard PAR lighting with three different UV-A intensities (6, 12, and 18 µmol m⁻² s⁻¹ at 385 nm) affected lettuce (Lactuca sativa) grown in a hydroponic system at DTU Lyngby over three weeks.

Method & measurements
Experimental setup
Lettuce seedlings were grown in a continuous-flow hydroponic system (Growstream 20) under DYNA Heliospectra LED lamps at 5700 K and 350 µmol m⁻² s⁻¹ PAR with a 16-hour photoperiod. Four batches received UV-A at 0 (control), 6, 12, and 18 µmol m⁻² s⁻¹. Opaque dividers prevented light contamination between sections.
Non-destructive biochemical measurements
All pre-harvest measurements used a CI-710s SpectraVue Leaf Spectrometer — a relatively new, non-destructive alternative to HPLC. Tracked indices included flavonoid reflectance (FRI), carotenoid index (CRI1), anthocyanin index (ARI1), chlorophyll A&B, and photochemical reflectance (PRI). Chlorophyll fluorescence (Fv/Fm) was measured with a Handy PEA fluorometer.
Data analysis
All spectral data, biomass results, and biochemical indices were processed and visualised in Python, including time-series barplots with 95% confidence intervals and image processing in Appendix B for leaf area estimation.
Key results
Shoot wet weight increased with UV-A dose: +1.98% (UV-6), +8.45% (UV-12), +16.0% (UV-18) vs control
Leaf area increased substantially: +14% (UV-6), +46% (UV-12), +65% (UV-18) vs control
Flavonoids elevated in all UV-A groups: +39.4% (UV-6), +12.63% (UV-12), +25.27% (UV-18)
Carotenoids increased dose-dependently: +8.80% (UV-6), +14.25% (UV-12), +16.21% (UV-18)
Photosynthetic efficiency (Fv/Fm) maintained or improved under UV-A, indicating no damaging stress at these intensities
Concluded that UV-A supplementation positively affects both biomass production and nutritional secondary metabolite content

Automatic Quality Control of Stream Water Level Time Series Using Deep Learning

MSc thesis investigating LSTM neural networks for automated anomaly detection and imputation in Danish stream water level data — developed in collaboration with SGAV and built around a custom synthetic anomaly framework for training data generation.

Background

Manual quality control (QC) of hydrological sensor data is resource-intensive and increasingly difficult to sustain at national scale. SGAV — the authority responsible for Danish stream monitoring — handles large volumes of water level time series that require review, correction, and imputation of anomalies such as spikes, offsets, drift, and baseline shifts. These anomalies arise from sensor malfunctions, frost events, aquatic weed growth, and field disturbances. This thesis investigated whether recurrent neural networks — specifically LSTM architectures — could automate parts of this workflow, reducing manual burden while maintaining the reliability needed for operational use.

Approach & models
Synthetic anomaly framework
A core contribution of the thesis. Since no labeled anomaly dataset existed, a framework was built to inject synthetic versions of five real anomaly types — spikes, offsets, drift, noise, and missing data — into clean time series. This enabled structured, repeatable model training and evaluation without requiring manual annotation of historical records.
LSTM 1 — Sequence-to-sequence with auxiliary inputs
A many-to-many LSTM that uses water level data from nearby hydrometric stations alongside precipitation and temperature as auxiliary inputs, predicting the full target sequence in one pass. Designed to leverage spatial correlation across the catchment. Achieved RMSE 141 mm and NSE 0.65 on the test set.
LSTM 2 — Autoregressive forecaster with memory protection
An autoregressive LSTM trained solely on target-station and weather data, predicting one step ahead and feeding its own output back as input. Augmented with memory protection — masking corrupted timesteps during backpropagation — and anomaly-aware loss weighting. Consistently outperformed LSTM 1, achieving RMSE 52 mm and NSE 0.95, though with a tendency to overfit to the injected synthetic anomalies.
Anomaly detection
Detection was implemented using a residual-based method with a static z-score threshold applied to the difference between model predictions and observations. LSTM 1 generated more true detections due to noisier outputs, but lacked the reconstruction quality needed for imputation. Detection sensitivity was limited, particularly for low-amplitude or gradual anomalies — reflecting constraints of the threshold methodology rather than model capability.
Key results
LSTM 2: RMSE 52 mm · NSE 0.95 on held-out test data
LSTM 1: RMSE 141 mm · NSE 0.65 — higher detection count but lower reconstruction quality
Synthetic anomaly framework successfully enabled training without labeled data
Memory protection and anomaly-aware loss weighting improved LSTM 2 robustness to corrupted inputs
Imputation quality sufficient for operational support; full autonomous detection remains an open challenge
Catchment map and data pipeline figures produced in Python and QGIS
